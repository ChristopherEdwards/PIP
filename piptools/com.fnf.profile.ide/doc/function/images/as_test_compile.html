PSL Test Compile

A Profile PSL procedure definition can be compiled on the host from Application Studio without saving the procedure to the Profile RTE. Compile errors are viewed in the Fidelity Console and in the Task pane. The contents must be saved locally before test compiling.

This function can be accessed from the right-click menu on the Resource Navigator in the Fidelity Profile Perspective. This function can also be accessed from the right click menu within the edit pane.

Fidelity Console

The Fidelity Console displays important messages about functions that have been run in Fidelity Application Studio. There are buttons to copy or erase all of the contents in the Fidelity Console.

Problems View

In addition to the information displayed in the Fidelity Console, compile warnings and errors create Problems in the Problems view. Double-clicking on Problems jumps the cursor to the location of the Problem. If the resource is not open in an editor, it will open in the default editor. Problems are removed after the problems in the code are fixed and the test compile function is re-run.

Help may be also be available by selecting the line of code in the edit pane and pressing Ctrl and the space bar. When this help is available, a pop up window is displayed with possible causes for the error and suggestions to fix it.
