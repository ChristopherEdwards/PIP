#IF/#ELSE and Literal Scope Variables

The simplest way to optimize code is to eliminate it. Use #IF/#ELSE blocks in the PSL source code to remove unnecessary code and conditional checks from the generated code and to optimize the run-time code path.

Use Literal scope variables to significantly reduce run-time indirection and execution, especially when referring to routine or function references in CUVAR or other static tables. In addition, Literal scope variables can make PSL source code more readable and maintainable by initializing a variable to a constant and using the more descriptive variable in the code. Because the literal value is compiled into the generated program, the variable reference (and the variable itself) is eliminated at runtime.
