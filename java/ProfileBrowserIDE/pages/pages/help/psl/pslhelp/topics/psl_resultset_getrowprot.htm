getRowProt Method

Class: ResultSet

Description

This method returns the protection value for the current row in the result set. To use this method, you must adhere to specific rules.

This method was created to facilitate the Profile v7.0 report writer conversion to PSL.

Syntax

ResultSet.getRowProt()

Parameters

None.

 

Returns

The concatenation of the column protection values of the individual columns of the row (as returned by ResultSet.getColProt()).

When Became Available

Profile v7.0

Examples

Profile's data item protection provides a framework, used by individual sites to impose access rules on data items. Since the implementation is the exclusive responsibility of the customer, the following examples only serve to demonstrate the interaction between data item protection logic and PSL code.

The examples assume that the following data item protection rules have been defined on the DEP file, and that the actual UserClass matches the specified value ("TLR"):

  • Tellers (denoted by %UserClass="TLR") are prohibited from seeing the balances (DEP.BAL) of employees (denoted by .DEP.CCODE="EMP").

  • Tellers have read-only access to the available interest accrued (DEP.AIACR) of employees.

Example #1 -- Discard Data Item Protection (by explicitly specifying "/PROTECTION=0")

  write "the userclass equals '", %UserClass, "'", !

  do colProt0("ACN") // shows all values listed in table

  do colProt0("BAL") // shows all values listed in table

  do colProt0("AIACR") // shows all values listed in table

  quit

 

 

public colProt0( String sNmOrNr)     // sample procedure

  type ResultSet rRs = Db.select("CID,ACN,BAL,CCODE,AIACR",

                                 "DEP","BAL > 10000",,,

                                 "/PROTECTION=0")

 

  if rRs.isEmpty() do {

       write "no records match your criteria",!

  }

  else  do {

       write "Column '",sNmOrNr,"' of matching records:",!

       while rRs.next() do {

              // getRowProt() always empty in this case

              write "row protection = '", rRs.getRowProt()

              write "', column protection ='"

              // getColProt() always empty in this case

              write rRs.getColProt(sNmOrNr)

              write "', and value ='", rRs.getCol(sNmOrNr)

              write "'", !

       }

  }

  quit

 

Example #2 -- Same as Example #1, but it uses "/PROTECTION=2"

  write "the userclass equals '", %UserClass, "'", !

 

  // ACN: shows all values, getColProt() always 0

  do colProt2("ACN")

 

  // BAL: shows refids A and C (with getColProt() = 0),

  //      empty string for refid B (with getColProt() = 3)

  do colProt2("BAL")

 

  // AIACR: shows refids A and C (with getColProt() = 0),

  //        shows refid B as well (with getColProt() = 2)

  do colProt2("AIACR")

  quit

 

 

public colProt2( String sNmOrNr)    // sample procedure

  type ResultSet rRs = Db.select("CID,ACN,BAL,CCODE,AIACR",

                                 "DEP","BAL > 10000",,,

                                 "/PROTECTION=2")

 

  if rRs.isEmpty() do {

       write "no records match your criteria",!

  }

  else  do {

       write "Column '",sNmOrNr,"' of matching records:",!

       while rRs.next() do {

              // getRowProt: shows "00000" for refids A and C

              //             shows "00302" for refid B

              write "row protection = '", rRs.getRowProt()

              write "', column protection ='"

              write rRs.getColProt(sNmOrNr)

              write "', and value ='", rRs.getCol(sNmOrNr)

              write "'", !

       }

  }

  quit
