Method Return Expressions

The simplest interface between the compiler and the method subroutine is for the subroutine to return the variable ‘return’ which PSL appends to the current M-generated code string variable ‘mcode’. Because methods usually involve multiple statements, lines, or blocks of logic, the return expressions usually take the form of a function or extrinsic function reference for assignment expressions. Void methods (i.e., those that do not return a value) usually return a subroutine label.

However, since PSL is dispatching to program logic, the method implementation code can manipulate any of the data structures used by the PSL compiler itself, and can register post-processing expressions for PSL to execute after it has passed through the PSL source. The method implementation logic may also generate functions or subroutines to append to the program, or manipulate variables that are used within the compiler itself.
