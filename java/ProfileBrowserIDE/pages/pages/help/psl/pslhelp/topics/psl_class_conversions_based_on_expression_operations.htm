Class Conversions Based on Expression Operations

The following rules apply to operations that occur between two different classes:

  • If the operators ' [ ] = < > are used, the expression returns a Boolean class.

       set x = a > b

  • If the oeprators * / \ # + - are used, the expression returns a Number class.

       set x = a * b

  • If the operator _ is used, the expression returns a String class.

       set x = a_b

  • If multiple operators are used, the expression evaluates from left to right.

       set x = a > b * c         // Returns a Number
       set x = a * b > c         // Returns a Boolean

  • If one operand is a Date class, the other operand is a Number class, and the operator is + or -, the expression returns a Date class.

       set x = %SystemDate + 1
       set x = %CurrentTime - 60

    This rule is recursive.

       set x = %SystemDate + 10 + 10

  • If both operands are Date or Time, the expression returns a Number class.

       set x = %Currentdate - %SystemDate

  • Methods use the class of the identifier.

       set x = "string"_123.justify(12,2)

    where the 123 identifier is a number. Therefore, in this example, the expression returns a Number class.

  • Use parentheses to control classes.

       set x = ("string"_123).extract(1,5)     // expression is a string

  • Methods can be applied directly to Strings, Numbers, [extrinsic] functions (as Strings), and variables.

       set x = "1,2,3".piece(",",2)
       set y = 123.4.justify(12,2)
       set z = $$^foo(par).extract(1,2)

  • M instrinsic functions are supported as methods for all primitive classes.
