Use of the #OPTIMIZE Compiler Command

The #OPTIMIZE command instructs the compiler to identify objects that are very local in scope (i.e., not passed as parameters) and to optimize the run-time code. In general, the object can be managed within private local variables instead of the object array, which appears to have a significant positive impact. The result is optimized result sets.

To enable this optimization, PSL code can use result sets in nested blocks. Passing a result set as a parameter would defeat the optimization. The generated result set code is restructured to place all the code related to a single Db.select cursor together in the generated program. In addition, the select statement is included in the open label to improve readability and debugging.

The default #OPTIMIZE action is ON. However, if #OPTIMIZE is not included in the PSL code, the optimizer will not be invoked.

This compiler command is optional in the current implementation. However, this command should be inserted in new PSL procedure and batch, as well as any existing procedures and batches that are modified. The generated code should be compared to a non-optimized version to validate the optimization. This process should be used for development in all versions.
