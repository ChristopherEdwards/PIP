Preference of Record<Class>.bypassSave Over Other Update Methods

The Record<Class>.bypassSave method avoids all schema logic related to table updates (other than index management and timestamps) and updates the database directly. This method can be significantly faster than Record<class>.save, Db.insert, and Db.update. However, only use the Record<Class>.bypassSave method in procedures where data validation and trigger logic can safely be ignored. While many procedures and batches can currently pass this safety test, schema logic can change and must be carefully reconciled with procedures that use bypassSave.
