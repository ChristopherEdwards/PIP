toRecord Method

Class: Row

Description

This method copies the properties of the Row object into the same property names in a Record<Class> object. This method inserts a Row class object into a RecordClass object based on matching column names. Multiple Row objects can be inserted into a Record object. Row objects can contain properties that are not mapped (i.e., no matching name or explicit map). Row columns that are named the same as a column in the Row object are moved to that location. All other columns must be manually defined in the columnMap parameter.

This method is useful for applications that import data from a file or message interface and subsequently store them in the database (e.g., conversion interfaces).

Syntax

Row.toRecord(Record rec,String columnMap)

Parameters

rec

The record to which the row is copied. The record must have been instantiated before it is passed to this method.

columnMap

The comma-separated list of column names to explicitly map between the Row class and the Record object. Each mapping is in the form NameInRow=NameInRecord.

If the NameInRecord portion is empty (e.g., NameInRow=), then the corresponding column in the Row object will not be mapped (ignored).

Returns

Void

When Became Available

Profile v7.0

Example

// uses table ZFSW04RW with columns XNAME,DNAME,DOB

type ResultSet rs = Db.select( "NAM,XNAME,DOB,TYPE", "CIF")

 

do Db.fastDelete("ZFSW04RW") // clear target table

 

while rs.next() do {

  type RecordZFSW04RW zfsw04rw = Class.new("RecordZFSW04RW")

  type Row rRow = rs.getRow()

  // translate Row to Record by mapping:

  // CIF.NAM   to ZFSW04RW.DNAME

  // CIF.XNAME to ZFSW04RW.XNAME

..// CIF.DOB   to ZFSW04RW.DOB

..//.and ignoring CIF.TYPE

  do rRow.toRecord(zfsw04rw, "DNAME=NAM")

  do zfsw04rw.bypassSave()

  write "copied ",rRow.xname," to ZFSW04RW",!

}
