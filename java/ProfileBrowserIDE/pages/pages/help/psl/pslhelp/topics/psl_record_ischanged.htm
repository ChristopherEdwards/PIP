isChanged Method

Class: Record

Description

This method determines whether a column has been changed, and whether the change was performed by user entry or by a programmer applying a business rule. This method relies on the setAuditFlag method being set in the label where the column has changed. The setAuditFlag method maintains the information in the object that is checked by the isChanged method. SQL builds the object with the audit information included.

 

This method is generally used in a trigger procedure definition.

Syntax

objectName.isChanged(String column, literal String changeType)

Parameters

column

The literal column or variable indicator of the column being checked.

changeType

The type of change to perform (e.g., USER, SYSTEM). All code generated within PSL defaults to SYSTEM-generated changes. USER changes are changes entered into the system through SQL commands. When SQL is called from internal routines, the programmer can add the SYSTEM-generated parameter to achieve the same characteristic of PSL.

Returns

1, if the value is changed.
0, if the value is not changed.

When Became Available

Profile v6.4

Example

type RecordDEP dep=Db.getRecord(“DEP”,“CID=:CID”)

do dep.setAuditFlag(1)

set dep.irn=5

 

//Literal column name

if dep.isChanged(“IRN”,“SYSTEM”)... //changed by system code

if dep.isChanged(“IRN”,“USER”)...   //changed by user

if dep.isChanged(“IRN”)...          //column was changed

 

* Variable CID must be defined prior to the getRecord statement.

* All of the if statements in this example are incomplete.
