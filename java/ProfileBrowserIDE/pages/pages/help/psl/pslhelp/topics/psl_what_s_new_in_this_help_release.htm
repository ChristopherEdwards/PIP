Use of ResultSets Versus DbSets

Enhancements to Generated Code for Result Sets

The following enhancements have been made to the generated code for result sets:

  • The object now contains an SQL representation of the db.select command in vobj(object,03) for results sets only.

  • Host variables are not protected with a $G when they are moved in the vobj array.

  • The test on the host variables being null, as well as the execution of $ZT, have been removed. Therefore, during execution of the code, undefined errors or null subscript errors are reported to the process user using the normal catch/trap code specified in the program. More errors may be logged in the error global, which provides a good mechanism to report bugs that reach the user community.

  • Date fields are no longer converted to Julian format.
