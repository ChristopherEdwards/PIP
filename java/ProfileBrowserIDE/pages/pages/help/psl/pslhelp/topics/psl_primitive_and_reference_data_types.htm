Primitive and Reference Data Types

Conceptually, there are two types of data in PSL:

  • Primitive types

  • Reference types

A primitive data type represents a single value. Examples of primitive data types are:

  • Blob

  • Boolean

  • Date

  • Memo

  • Number

  • String

  • Time

Reference data types contain values that point to or identify objects such as a RecordDEP object or a ResultSet object. The name of the class is used as the type. When you declare a variable using the name of a class for its type, the variable will contain a reference to an object of that class.

A Type enclosed in braces {} specifies a type cast operation. A cast always produces a pure value of the specified type, by converting its operand to that type if necessary. A cast can convert between certain primitive types. A cast between reference types never alters the type or content of the object.

PSL supports specific operators (+ - * / \ # _ ‘ = > < [ ] ) on primitive data types. However, PSL does not support these operators on reference data types. Attempting to use these operators on reference data types will produce a compile time error.
