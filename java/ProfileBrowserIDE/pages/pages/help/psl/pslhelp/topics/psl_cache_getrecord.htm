getRecord Method

Class: Cache

Description

This method retrieves a single record from within a process’ local memory, calling the Db.getRecord method itself if a record does not exist in the cache. A Cache object is stored within its identifer. If the identifier is an array, the cache is stored at the declared subscript level.

This method instantiates an object; the system copies the record from the database incrementally as it references its properties. Use this method to create a new object in modify mode.

To implement the Cache class, replace references to Db.getRecord() with Cache.getRecord.

To retrieve multiple records simultaneously, use the Db.select method. To define a new object in create mode, use the Class.new method.

Syntax

Cache.getRecord(literal String table, literal String accessKeys)

Parameters

table

A valid database table name.

accessKeys

A comma-separated list of access keys for the table. This parameter can also specify an assignment (e.g., "ACN=:CIFNUM") or a column reference (e.g., DEP.ACN).

 

Note that the Cache.getRecord does not include a third parameter (classNew) as the Db.getRecord method does, because the Cache class only accepts records that currently exist in the database.

Returns

An object of type RecordTABLE (e.g., RecordDEP, RecordLN).

When Became Available

Profile v6.3

Example

// Table with one key

 

type public Cache %CACHE()

type RecordCIF cif1 = %CACHE("CIF").getRecord("CIF","ACN=:ACN")

type RecordCIF cif2 = %CACHE("CIF").getRecord("CIF",ACN=:dep.acn)

 

// Table with multiple keys

 

type public Cache %HIST

type RecordHIST hist =

%HIST.getRecord("HIST","CID=:CID,TSEQ=:TSEQ")
