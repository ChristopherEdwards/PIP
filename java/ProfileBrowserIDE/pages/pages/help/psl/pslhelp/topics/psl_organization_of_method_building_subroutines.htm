Organization of Method-Building Subroutines

Method-building subroutines should be organized within routines that refer to the class of the method. As a convention, prefix all method routines with the characters ‘UC’. For example, all the ResultSet methods are in a program called UCRESULT, and all the Db methods are contained in a routine call UCDB.

If possible, name the subroutine the same as the method. For example, the ResultSet.getRow method is mapped to the entry reference getRow^UCRESULT. This naming convention may not be always be possible due to M naming restrictions (which is why the routine in this example is UCRESULT instead of UCRESULTSET). However, try to use an intuitive or logical name.

Prefix the names of custom methods in the OBJECTMET table and their associated library program with the sentinel character ‘Z’ (e.g., library program "ZUC*").

For methods that return values, ensure that the method does not modify the last truth value (i.e., $T).
