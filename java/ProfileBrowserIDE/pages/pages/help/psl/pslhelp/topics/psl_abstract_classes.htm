Abstract Classes

Object-oriented design enables you to define a superclass that declares the structure of a class without providing a complete implementation of every method. You may want to define a superclass that only defines a generic form that is shared by all its subclasses, yet enabling each subclass to specify the details. For example, the PSL Record class contains all the methods used by its subclasses RecordACN, RecordDEP, RecordLN, RecordCRCD, etc. You cannot create an instance of the Record class because it has no meaning. However the Record class does contain the methods of all its subclasses. Classes defined in this way are called abstract classes.

If you want to require that specific methods within a superclass be overridden by the associated subclasses, define the superclass methods as abstract methods. Such methods have no implementation within the superclass but are important to the subclasses. Any class that contains one or more abstract methods must also be declared abstract.
