Non-Object Reference Variable Scope in PSL

Non-object reference variables passed to a procedure can be referenced, updated, and destroyed within that procedure. PSL follows the M standard. However, to improve source code readability and intent, variables that are shared between processes should be passed as parameters whenever possible.

An important exception to the variable handling is the interface to PSL methods. PSL assumes that all variables referenced by a method at run-time are passed as parameters (or at a minimum not changed within the method). That is, PSL expects the value of any variable that is not a parameter to return with the same value it had when the method was invoked.
