copy Method

Class: Record

Description

This method creates a new Record (i.e., toRecord) that is a copy of the original Record (i.e., fromRecord).

You can only copy like Records:

  • You can only copy multi-node records to other records of the same class. That is, you can only copy objects of the RecordACN class to other objects defined for that class. You cannot copy objects of the RecordACN class to objects of the RecordLN class.

  • A single node record can always be copied to another record of the same class.

  • In addition, you can copy a single node record of one class to a single node record of another class. In that case, you must provide a mapping of columns that appear in the two objects that are not named the same but are equivalent. For example, if copying from TTX to HIST, ttx.tim is equivalent to hist.time. The mapping consists of toColumn=fromColumn pairs, separated by commas. An SQL literal (i.e., enclosed in single quotes) or :HostVar may be used instead of fromColumn.

This method overrides Reference.copy() for all Record descendant classes.

Syntax

type RecordXXX toRecord = fromRecord.copy(literal String colMapping)

Parameters

colMapping

A literal string that defines the mapping of columns that are not named the same between objects but are equivalent. This parameter is only used when trying to copy a single node record from an object of one Record class to another Record class.

Returns

A new object of the same type as the original object.

When Became Available

Profile v6.0

Profile v7.0 - Removed mode parameter as the Record.setMode method performs that function. Added colMapping parameter.

Example

type RecordHist hist

set hist = ttx.copy("time=tim,brcd=brcde,uid=uidt,

xhs16=lnerc,ben=cc")

set hist.tseq = %TRNHSEQ

sethist.tjd = %SystemDate

do hist.bypassSave()
